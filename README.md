# Object-head-Museum
一个异头社区。
异头收藏馆 - 微信小程序 README

基于 Vue3 + uni-app 开发的二次元虚拟形象专属分享社区，专注于异头、覆面、OC、虚拟头像的公开分享与互动，严格遵循微信小程序规范及相关法律法规，支持个人主体合规上架，可作为前端/小程序开发求职项目展示。

一、项目定位

面向二次元爱好者，打造纯公益、无经营性质的虚拟形象分享社区，核心聚焦「头像、异头、覆面、OC、虚拟形象」的图文展示与公开互动，全程规避高风险功能，确保合法合规、顺利过审。

✅ 微信小程序类目：文娱 - 图片/音频 或 社交 - 社区（均支持个人主体上架）

二、合规规范（核心风险规避，重中之重）

严格遵循以下规范，彻底规避法律、刑事及平台违规风险，确保项目安全上架及运营：

- 无私密互动：仅支持公开发帖、评论、点赞、收藏，不做站内信、私聊等一对一互动，规避帮信罪、诈骗工具相关风险。

- 无经营行为：全程免费，无支付、无打赏、无会员、无虚拟币，符合个人主体小程序运营要求，规避非法经营风险。

- 无地理位置相关功能：不获取用户位置信息，不做同城、附近的人等匹配功能，规避涉黄、涉诈相关风险。

- 内容严格管控：仅允许发布二次元异头、覆面、OC、虚拟头像相关内容；禁止发布真人照片、露脸内容、低俗及性暗示内容，规避传播淫秽物品相关风险。

- 内容安全审核：接入微信官方内容安全API，对所有发布的图片、文本进行自动校验，确保内容合规，降低平台及开发者责任风险。

- 完善合规功能：内置举报、屏蔽、删除功能，支持用户举报违规内容、屏蔽违规账号，是微信小程序过审硬性要求。

- 用户隐私保护：仅使用微信授权登录，不额外收集用户手机号、身份证等敏感信息，规避侵犯公民个人信息罪风险。

- 无诱导行为：不设置强制分享、强制关注等功能，不违反微信小程序平台运营规则，避免项目被下架。

三、技术栈

贴合前端/小程序开发求职需求，选用主流、实用的技术栈，覆盖核心知识点，可直接在面试中重点阐述：

- 核心框架：Vue3 + <script setup> 组合式 API

- 跨端框架：uni-app（适配微信小程序，支持快速编译上线）

- 状态管理：Pinia + 本地缓存持久化（用户信息、收藏数据等）

- UI组件库：uview-plus / uv-ui（小程序专用，提升开发效率）

- 网络请求：uni.request 封装 + 响应拦截（统一处理请求、错误）

- 图片处理：多图上传、图片压缩、预览、懒加载（优化用户体验）

- 安全校验：对接微信内容安全API（msgSecCheck文本校验、imgSecCheck图片校验）

- 工程优化：ESLint 代码规范、小程序分包加载、防抖节流（提升性能）

四、核心功能（完整覆盖需求，适配面试展示）

1. 首页

- 采用瀑布流布局，展示异头、覆面等虚拟形象图片，视觉体验贴合二次元风格

- 分类筛选：支持异头、覆面、OC、暗黑系、可爱系五大分类，快速定位内容

- 排序功能：支持热门、最新、收藏量三种排序方式，满足用户浏览需求

- 下拉刷新、上拉加载：优化大数据量展示体验，避免页面卡顿

2. 发布页

- 多图上传：支持最多9张图片上传，适配虚拟形象多图展示需求

- 图片处理：自动压缩图片，减少加载耗时；支持上传前预览、删除

- 内容编辑：支持输入标题、发布文案，选择对应分类标签（精准归类）

- 安全校验：发布前自动调用微信内容安全API，校验图片和文本，违规内容禁止发布

3. 详情页

- 图片查看：支持单张图片放大查看，引导用户通过系统相册保存图片（合规保存方式）

- 互动功能：支持点赞、收藏，评论盖楼（多级评论），实现公开互动需求

- 合规功能：内置举报按钮，用户可举报违规帖子、评论，后台可及时处理

4. 个人中心

- 个人数据管理：展示我的发布、我的收藏、我的喜欢，支持编辑、删除自己的发布内容

- 个人信息设置：支持修改昵称、头像（同步微信授权信息，不额外收集数据）

- 合规必备：提供用户账号注销功能，注销后15个工作日内删除用户相关数据，符合《个人信息保护法》

- 合规说明：内置隐私政策、用户协议、关于页面，明确平台责任与用户义务

5. 简易后台管理

- 帖子审核：对用户发布的所有帖子进行人工二次审核（配合机器自动校验），违规帖子可下架、删除

- 评论管理：查看所有用户评论，删除违规评论，屏蔽恶意评论内容

- 用户管理：支持对违规用户进行禁言操作，限制其发布、评论权限

- 屏蔽词管理：可添加、编辑屏蔽词库，自动拦截包含违规词汇的文本内容

六、运行与上线说明

1. 开发环境：使用 HBuilderX 打开项目，安装相关依赖（npm install）

2. 小程序配置：在 manifest.json 中配置个人微信小程序 AppID，选择对应类目（文娱-图片/音频 或 社交-社区）

3. 本地运行：运行到微信开发者工具，调试页面功能、接口对接及合规校验逻辑

4. 上线审核：完善隐私政策、用户协议页面，上传代码提交微信小程序审核，审核通过后即可上线
